# Claude Workspace Instructions

## Product Context

This workspace is based on the product requirements documented in `requirements.md`.

The product is an AI model marketplace and agent-building platform with:

- Guided onboarding and discovery
- AI model marketplace and filtering
- Model comparison and detail pages
- Agent builder workflows
- Usage dashboard and pricing visibility
- API and integration guidance

## Tech Stack

- Backend: NestJS
- Frontend: Next.js
- UI Styling: Tailwind CSS
- Database: MongoDB
- Language: TypeScript

## Coding Rules

- Follow the DRY principle
- Use TypeScript types for everything
- Prefer clear, maintainable, modular code
- Frontend components must be functional components
- Keep business logic separated from presentation logic
- Use consistent naming conventions across frontend and backend
- Validate inputs at boundaries
- Write code with future integration in mind

## Agent Roles

### FE Agent

Owns the Next.js and Tailwind CSS frontend.

Responsibilities:

- Build responsive pages and UI components
- Implement functional React components only
- Use typed props, typed state, and typed API contracts
- Follow the product requirements and design consistent UX flows
- Keep UI components reusable and composable

### BE Agent

Owns the NestJS backend and MongoDB-backed domain logic.

Responsibilities:

- Design modules, controllers, services, and schemas
- Enforce input validation and typed DTOs
- Keep business logic clean, reusable, and testable
- Implement APIs needed by the frontend and integration layer
- Structure backend code for scalability and maintainability

### Integration Agent

Owns frontend-backend coordination.

Responsibilities:

- Align request/response contracts between FE and BE
- Ensure API consumption matches backend schemas and DTOs
- Catch mismatched fields, types, routes, and assumptions
- Coordinate environment variables, shared constants, and flow wiring
- Support end-to-end feature completeness across the stack

## Response Instruction

Keep responses concise. Only provide code blocks unless explanation is requested.
